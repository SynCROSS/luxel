# Benchmarks

Luxel micro-benchmarks and future competitor scorecard fixtures.

## Horizon C2 skeleton

- **Registry:** `fixtures/micro/{counter,list,table}` — counter runnable; list/table reserved (`pending`).
- **CLI:** `luxel bench` emits JSON lines: `{ "fixture", "metric", "value" }`.
- **Metrics (counter):** SSR throughput (req/s), client JS bytes.
- **Competitor runners:** wired in `@luxel/luxel` bench registry (counter SSR); WinRK harness uses `@luxel/luxel/bench` public seam.
- **Deferred until v1.0 (C4g):** weighted scorecard; app fixtures (blog/dashboard/ecommerce/auth); server-stress fixtures; full §11 matrix exit. **v1.0 exit:** full §11 matrix per root `CONTEXT.md` **Benchmark scorecard (C4g)**.
- **Idiomatic peak rows:** CSR + SSR competitors use reactive state at framework fastest idioms; one shared component per stack (CSR build + SSR import) — see root `CONTEXT.md` **Idiomatic peak benchmark row** and `docs/benchmarks/fairness.md`.
- **Source of truth:** competitor components + SSR render fns in `@luxel/luxel/bench` (`packages/luxel/src/bench/competitors/`). WinRK servers import bench seam; `packages/bench/competitors/` = CSR Vite dist builds only. CSR build sync uses `competitorSource()` from `@luxel/luxel/bench` — no relative `../../luxel` paths.
- **Worker-pool rows vs SynCROSS:** Luxel pools: **parent-once compile** (`precompile-luxel-bench.ts` → `bench-pool.json` + `server/{slug}/server.mjs`); workers **hydrate/import-only**; counter pool passes **`pooledHtml`** so workers **ack without cloning HTML** on the hot path. Framework inline pools: `precompileInlineSsr()` + worker `importPrecompiled*()`. **Worker sizing (fixture-specific):** counter `*-worker-pool` rows default to **1 worker** (WinRK-tuned parity gate: `react-ssr-worker-pool` ≥ `react-ssr` RPS); spiral `*-worker-pool` rows use full **`hardwareConcurrency`** (`benchRenderWorkerCountForFixture("spiral")`); env override `BENCH_RENDER_WORKER_COUNT`. **Dispatch:** SynCROSS round-robin FIFO; single-worker concurrent jobs must not emit `MaxListenersExceededWarning`. **Worker backend:** Unix default **Bun Worker**; **Windows default `node:worker_threads`** (`BENCH_RENDER_WORKER_BACKEND` pinned when unset — node passes react counter parity gate; Bun on win32 may trail inline). Override `BENCH_RENDER_WORKER_BACKEND=bun|node`. **fastify-html `*-worker-pool`:** Fastify listen + round-robin worker pool (SynCROSS `fastify-html-bun` / `fastify-html-workers`). Luxel pool HTTP: **Bun.serve**. External Luxel `*-worker-pool` rows: child-server subprocess + **`warmupBenchUrlBurst`** before load test.
- **WinRK fair-run scope (locked):** inline Bun.serve + Luxel worker-pool rows must be green before geo-mean; prod-stack `pending` / error rows excluded from denominator (`fairness.md`) — fix Next/SolidStart/SvelteKit pools in follow-up, not blocking inline fairness.
- **`bench:repro` exit gate (locked):** every stack in fixture must run; every `role: framework` row must be `status: ok` with **0 raw winrk errors** — `pending` / `error` / incomplete fixture → **exit 1** (`src/winrk/repro-gate.ts`, env `BENCH_REPRO_GATE=1` set by `bench:winrk:repro-all` only). Bisect via `bench:winrk` + `WINRK_STACK` skips gate.
- **WinRK stack reliability:** After high-concurrency load-test windows, Windows can transiently refuse new localhost client sockets (`Unable to connect`). Harness uses `benchFetch` (Connection: close + retry), `waitForServerReady` before probe, **`BENCH_POST_WINRK_COOLDOWN_MS`** (default **5000** on Windows, **1000** elsewhere) + `waitForServerReady` before post-winrk latency sample, `BENCH_STACK_COOLDOWN_MS` (default **10000** on Windows, **2000** elsewhere) between stacks, `BENCH_STACK_RETRY_ATTEMPTS` (default 3) for retriable connect/parse failures, and `Bun.gc(true)` between stacks when available. **Load tester:** `BENCH_LOAD_TESTER` = `auto` (default) | `winrk` | `bombardier` — auto picks bombardier when on PATH, else winrk. On nonzero winrk errors, harness **retries same stack with bombardier** (`-c400 -d15s -l <url>`) when available. Inline HTTP: **Bun.serve** (`BENCH_FORCE_NODE_HTTP=1` opt-in only). **Post-winrk latency:** `BENCH_LATENCY_SAMPLES` default **1000** on Windows / **5000** elsewhere; **`0`** = winrk histogram only (p95/p99 from winrk max). **Measurement model:** **one stack server at a time** — start → load test → stop; never concurrent stack servers or overlapping load windows. `bench:repro` = **one Bun subprocess per fixture** (counter, spiral), not per stack; opt-in `BENCH_ISOLATE_STACKS=1` only for bisect/crash recovery (spawns per-stack subprocess — slow, many processes). `bench:cleanup-orphans` kills stale repro chain (`bench:repro`, `winrk-repro-all`, `winrk-run`) and stack children (`stack-child-server`, `winrk-isolated-stack`) from aborted runs. **`*-worker-pool` rows:** skip redundant `warmupBenchUrl` + post-winrk 5k latency sample (winrk stats only). Optional `BENCH_STACK_TIMEOUT_MS`. **React inline SSR:** `NODE_ENV=production` before parent imports compiled `react` artifact (`loadReactTsxForSsr`); `ssr-render.ts` imports React lazily. **React inline worker pools:** lazy `@luxel/luxel/bench/ssr` import in WinRK inline servers (parent `react-dom/server` poisons workers); react render workers load `react-dom/server` before `react`, skip parent `bench-env` auto-hook, cache-bust only when named exports are empty. Registry runs framework `*-worker-pool` rows before inline `*-ssr` for same framework. Prod-stack worker pools (`react-rsc`, `solidstart`, `sveltekit`) export `startBenchWorkerPoolServer` from `.bench-server.mjs`.
- **Framework prod stacks:** SvelteKit / SolidStart / Next RSC counter routes sync shared `competitorSource()` (reactive idiomatic peak — not literal static `0`). **SvelteKit Windows build:** `competitors/build/sveltekit-ssr.ts` runs `vite build` in **Node subprocess** (not Bun), serializes via `.svelte-build.lock`, retries on transient `EUNKNOWN` `.svelte-kit/generated/server/internal.js` lock (Kit prerender worker re-sync). Benign `Falling back to loading svelte.config.js` may log; hard fail → retry (8×).
- **WinRK metrics:** RPS (winrk), p50 (winrk median + sample), p95/p99 (post-winrk fetch sample), CPU/memory time-series during winrk, response bytes (HTML weight).
- **Stack wall-clock vs winrk duration:** `WINRK_DURATION` (default **15s**) is only the load-test window — **not** per-stack total time. Each non-`worker-pool` row also runs **250-req JIT warmup** + **post-winrk latency sample** (after cooldown) + **2s** inter-stack cooldown. Pre-fix Windows runs could spend **~5–6 min**/stack on the 5k sample when localhost sockets were still exhausted; harness now cools down first and defaults to **1k** samples on Windows. `*-worker-pool` rows skip warmup + latency sample. **`bench:repro`** = competitor builds + luxel-core build + 53 sequential stacks. **Fast bisect:** `bench:winrk` + `WINRK_STACK` + env overrides; skip rebuild with `bench:winrk:repro-all` when dists exist.
- **Dashboard winners:** flat across visible framework rows per fixture; baselines hidden by default (`role: baseline`). No per-tier or per-mode winner chips — user filters/sorts table manually (inline vs prod-stack labels disambiguate). Matches **flat geo-mean** gate policy in `docs/benchmarks/fairness.md`.
- **WinRK fixtures:** counter (35 stacks) + spiral (18 stacks). Luxel counter rows: `luxel-ssr` (TS default), `luxel-ssr-full`, `luxel-ssr-native`. Luxel spiral rows: `luxel-spiral-ssr` (TS default), `luxel-spiral-ssr-full`, `luxel-spiral-ssr-native`. **No `WINRK_FIXTURE=all`** — run counter and spiral as separate Bun processes; merge with `bench:winrk:merge` → `winrk-all-latest.json` (dashboard also loads per-fixture JSON). Counter has two SSR deployment tiers (inline Bun.serve vs prod-stack Node) — labels disambiguate; **geo-mean flat** across all framework rows per fixture. **Bisect:** `WINRK_STACK=id` (comma list) or `WINRK_STACK_UNTIL=id` (registry-order prefix) on `bench:winrk` only — `bench:repro` / `bench:winrk:repro-all` **strip** those filters so full matrix runs. Optimization notes in `src/winrk/optimizations.ts` + `docs/benchmarks/stacks.md`, embedded as `optimizations[]` in JSON.
- **WinRK runbook:** Fixture artifacts land in `docs/benchmarks/runs/winrk-latest.{json,md,jsonl}` (counter) or `winrk-spiral-latest.*` (spiral). Per-stack observability JSONL: `docs/benchmarks/runs/stacks/<fixture>/<stackId>.jsonl` — fixture start wipes `stacks/<fixture>/`; one line per stack per run; ok rows carry parsed metrics + load-tester `raw`. **stderr progress:** interactive TTY shows `[n/N] <id>: running…` then overwrites same line with final `ok`/`pending`/`error` summary; non-TTY logs final line only. `wrote docs/...` once at fixture end (incremental checkpoint writes are silent). **stdout:** no bench output on normal `bench:winrk` runs — machine-readable rows live in fixture JSON/JSONL + per-stack files only (`winrk-isolated-stack` subprocess still emits one stdout line for parent IPC when `BENCH_ISOLATE_STACKS=1`). **Bisect:** `WINRK_STACK` / `WINRK_STACK_UNTIL` infer fixture (overrides stale `WINRK_FIXTURE`); `WINRK_FIXTURE=all` without stack filter is fatal — use `bench:winrk:repro-all`. Orphan cleanup after aborted runs: `bun run bench:cleanup-orphans` (also runs at `bench:winrk` start/end). Registry source of truth: `src/winrk/registry.ts`; drift guard: `src/winrk/registry-truth.test.ts`.
- **Dashboard:** React app at `packages/bench/dashboard/` — post-run table/charts, optional JSON file watch. Counter rows ship live click handlers (luxel hydration or shared inline SSR script); see `docs/benchmarks/fairness.md`.
